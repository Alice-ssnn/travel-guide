# 瑞士法国旅行指南设计规范

## 项目概述

### 项目目标
创建一个瑞士法国旅行指南的HTML/CSS/JS应用，用于旅行途中实时参考。基于提供的12天详细行程文档，提供每日时间线、交通路线导航、地图显示、景点详情和快速搜索功能。

### 使用场景
1. **旅行途中实时参考**：边走边看，作为"参考手册"/"路线指南"
2. **移动端优先**：手机为主要使用设备，同时适配电脑查看
3. **离线能力**：网络不稳定时仍可查看行程数据
4. **静态行程**：行程不需要实时更新，基于预定义的12天行程

### 核心需求
1. **每日行程时间线**：最重要，按时间顺序展示全天活动
2. **交通方式和路线导航**：显示行程间的交通方式、路线、预计时间
3. **地图显示**：交互式在线地图，显示完整路径，可跳转到Google Maps App
4. **快速查询**：搜索特定信息，如景点、住宿、交通
5. **景点详细信息**：景点介绍、访问方式、营业时间、购票方式

## 信息架构

### 整体架构（方案A确认）
- **主页**：卡片式概览，显示12天行程摘要
- **单日详情页**：详细时间线视图，可切换地图/交通/详情视图
- **搜索功能**：全局搜索栏，支持关键词搜索和筛选
- **收藏功能**：可收藏重要地点或活动
- **地图视图**：交互式地图显示完整行程路线

### 导航流程
```
主页（12天卡片概览）
    ↓ 点击某天卡片
单日详情页（时间线视图）
    ↓ 分段控制切换
    ├── 时间线（默认）
    ├── 地图视图
    ├── 交通详情
    └── 详细信息
    ↓ 点击活动
活动详情模态框/页面
    ↓ 地图按钮
Google Maps App导航
```

## 用户界面设计

### 设计风格
- **苹果设计理念**：简约清晰，充足留白，圆角设计，系统字体
- **色彩系统**：
  - 主色调：蓝色系 (#667eea → #764ba2 渐变)
  - 辅助色：橙色 (#ff9500) 表示重要/提醒，绿色 (#30d158) 表示进行中/成功
  - 中性色：#8e8e93 次要文本，#f2f2f7 背景
- **字体**：-apple-system, BlinkMacSystemFont, sans-serif
- **触摸友好**：按钮大小≥44px，卡片间距充足

### 主页设计（选项A确认）
- **布局**：垂直滚动卡片列表，每张卡片代表一天
- **卡片内容**：
  - 天数标识（第1天，第2天...）
  - 日期（4月24日 周五）
  - 城市/地点标签（颜色编码）
  - 当天关键活动摘要（图标+文字）
  - 标签分类（飞行、自驾、酒店、湖泊等）
- **顶部工具栏**：
  - 行程标题"瑞士法国之旅"
  - "今天"按钮快速定位当前日期
  - 搜索入口

### 单日详情页（改进版确认）
- **页面头部**：
  - 渐变背景色，显示天数、日期、标题
  - 分段控制：时间线（默认）/地图/交通/详情
  - 当天状态标识（如"今天"）
- **当天概览卡片**：
  - 住宿信息（酒店名称、入住时间）
  - 交通方式（自驾租车、租车公司、费用）
  - 重要提醒（租车需国际驾照、山区驾驶注意等）
- **时间线视图**：
  - 按时间顺序排列活动（8个活动·12小时）
  - 每个活动卡片包含：
    - 时间标签（07:00）
    - 活动图标和标题（🍽️ 酒店早餐）
    - 活动描述（欧陆式自助早餐）
    - 地点信息（酒店名称、地址）
    - 持续时间标签（30分钟）
    - 分类标签（餐饮、费用等）
    - 操作按钮（详情、地图、导航）
  - **行程路线清晰显示**：
    - 起点→终点箭头布局
    - 交通方式图标和说明
    - 距离和预计时间
    - 路线步骤分解（如"A1→A6→A8"）
- **底部工具栏**：
  - 收藏本日
  - 打开地图
  - 分享行程

### 其他视图（通过分段控制切换）
1. **地图视图**：
   - Google Maps嵌入
   - 显示全天行程路线
   - 地点标记（点击跳转到Google Maps App）
   - 路线显示、地点标记、导航跳转

2. **交通详情视图**：
   - 交通方式汇总
   - 自驾路线详情、租车信息、停车位置
   - 油费预估、公共交通选择

3. **详细信息视图**：
   - 景点介绍、开放时间、门票价格
   - 住宿详情、餐厅推荐、实用贴士

## 功能规格

### 时间线功能
- **时间顺序展示**：全天活动按时间排序
- **时间花费显示**：每个活动预计持续时间
- **路线清晰化**：明确显示行程转移（如"酒店→机场"）
- **状态指示**：已完成/进行中/未开始（颜色编码）
- **快速操作**：每个活动可查看详情、地图、导航

### 地图功能
- **Google Maps集成**：
  - 嵌入Google Maps JavaScript API
  - 显示完整行程路线
  - 标记所有关键地点（住宿、景点、交通枢纽）
- **导航跳转**：
  - 点击地点标记跳转到Google Maps App
  - 提供多种交通方案选择（驾车、公共交通、步行）
- **路线显示**：
  - 绘制当天行程路径
  - 显示距离和预计时间
  - 支持缩放和拖拽

### 搜索功能
- **全局搜索**：顶部搜索栏
- **搜索范围**：
  - 景点名称
  - 住宿信息
  - 交通方式
  - 活动描述
- **筛选功能**：
  - 按类型筛选（餐饮、景点、交通、住宿）
  - 按费用筛选
  - 按城市/地区筛选
- **搜索结果**：直接跳转到相关活动或详情页

### 收藏功能
- **收藏项目**：重要地点、必去景点、推荐餐厅
- **收藏列表**：单独页面查看所有收藏
- **离线访问**：收藏内容优先缓存

### 离线功能
- **PWA支持**：渐进式Web应用，可安装到主屏幕
- **Service Worker缓存**：
  - 行程数据缓存
  - 地图静态图缓存（网络不稳定时显示）
  - 重要图片资源缓存
- **IndexedDB存储**：
  - 搜索索引
  - 收藏数据
  - 用户设置

## 技术架构

### 前端技术栈
- **核心**：HTML5, CSS3, JavaScript (ES6+)
- **UI框架**：纯CSS实现，无外部框架依赖
- **地图**：Google Maps JavaScript API
- **图标**：Unicode emoji + 少量SVG图标

### 数据转换
- **源数据**：提供的markdown行程文档（/Users/zsn/Downloads/🌟 瑞士·南法·巴黎12日深度游完整攻略 (1).md）
- **转换策略**：
  1. 手动提取关键信息到JSON数据结构
  2. 补充地理坐标（通过Google Maps Geocoding API或手动查找）
  3. 验证数据完整性和一致性
- **数据更新**：行程数据为静态，更新时需要重新生成JSON文件

### Google Maps集成
- **API密钥**：需要配置Google Maps JavaScript API密钥
  - 获取方式：通过Google Cloud Console创建项目，启用Maps JavaScript API，生成API密钥
  - 使用限制：需配置HTTP referrer限制，建议限制为应用域名
  - 费用：每月前$200额度免费（约28,000次地图加载）
- **地图功能**：
  - 显示多个标记点（住宿、景点、交通点）
  - 绘制折线连接路线
  - 信息窗口显示地点详情
  - 点击标记跳转到Google Maps App
- **导航跳转**：
  ```javascript
  // 跳转到Google Maps App
  function openInGoogleMaps(latitude, longitude) {
    const url = `https://www.google.com/maps/dir/?api=1&destination=${latitude},${longitude}`;
    window.open(url, '_blank');
  }
  ```

## 风险与缓解

### 技术风险
1. **Google Maps API限制**：
   - 风险：API调用次数限制，费用问题
   - 缓解：使用静态地图备用，合理控制API调用频率

2. **离线地图准确性**：
   - 风险：离线时地图数据可能过时
   - 缓解：提供离线路线说明文字，重要地点截图

3. **浏览器兼容性**：
   - 风险：某些API在旧浏览器不支持
   - 缓解：渐进增强，基础功能确保兼容

### 用户体验风险
1. **移动端操作复杂**：
   - 风险：功能太多导致界面复杂
   - 缓解：遵循移动端设计最佳实践，简化操作流程

2. **离线功能理解**：
   - 风险：用户不理解离线功能如何使用
   - 缓解：清晰的离线状态提示，简单明了的说明
